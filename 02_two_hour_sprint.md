# 2 小时冲刺计划

目标：在 2 小时内产出一个可报名的 60 分版本，并留下冲一等奖的升级路线。

## 0-15 分钟：补齐规则与定题

- [x] 读取第二篇征集文章。
- [x] 补第一篇文章规则。
- [x] 抽取报名表字段。
- [ ] 截图确认精确截止日期、获奖公布日期。
- [x] 定暂定 Skill 名称：城市踏勘官 CityScout。

产物：

- `00_rules.md`
- 最终题目一句话

## 15-35 分钟：写作品核心说明

- [ ] Skill 名称
- [ ] 使用场景
- [ ] 目标用户
- [ ] 痛点
- [ ] 高德能力调用点
- [ ] 功能描述

产物：

- `submission/skill_brief.md`

## 35-70 分钟：做 Demo 骨架

优先方案：一个本地 HTML Demo。

- [ ] 输入区：城市/片区、任务类型、时间预算、出行方式。
- [ ] 输出区：踏勘路线、点位清单、观察任务、汇报摘要。
- [ ] 高德能力说明区：每一步对应哪个开放平台能力。
- [ ] 示例数据：先用一个真实片区生成可信结果。
- [ ] README 页面突出 GitHub 收藏和 ClawHub 下载路径。

产物：

- `demo/index.html`

## 70-95 分钟：写使用教程与分享代码

- [ ] 使用步骤。
- [ ] API 能力说明。
- [ ] 输入输出示例。
- [ ] 代码/伪代码。

产物：

- `submission/tutorial.md`
- `submission/code.md`

## 95-115 分钟：打磨成可提交材料

- [ ] 报名表可复制版本。
- [ ] 技术社区文章标题和摘要。
- [ ] Demo 截图/演示说明。
- [ ] 原创声明与知识产权说明。

产物：

- `submission/form_answer.md`
- `submission/social_post.md`

## 115-120 分钟：提交前检查

- [ ] 是否包含 Skill 名称、使用场景、功能描述、使用教程、分享代码。
- [ ] 是否明确调用高德开放平台能力。
- [ ] 是否有 Demo 演示。
- [ ] 是否避免与官方示例同质化。
- [ ] 是否有社交传播标题/tag。

## 需要 Frank 现在补的最小信息

请优先给我第一篇文章正文或截图，尤其是以下部分：

- 评审标准
- 提交格式
- 截止日期
- Demo/代码要求
- 是否有主题限制

如果暂时不给，我们也能继续先做 `城市踏勘官` 的 MVP。
