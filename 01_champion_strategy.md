# 冠军策略

## 判断

要冲一等奖，不能只做“好玩的地图推荐”。冠军方案应同时满足五件事：

1. 高德能力不可替代：路线、POI、地理编码、距离、行政区、周边搜索、路径规划等能力必须进入核心流程。
2. 真实痛点强：最好是高频、高价值、普通推荐类产品做不好的问题。
3. Demo 可演示：评委能在 1-3 分钟内看懂输入、地图能力、输出结果。
4. 传播友好：标题一眼能懂，截图/视频能展示结果。
5. 可安装可收藏：报名表将 MVP 奖与 ClawHub 下载量、GitHub 收藏量绑定，所以仓库包装不能弱。

## 我建议的冠军方向

首选：城市更新/选址/现场踏勘助手

暂定 Skill 名称：`城市踏勘官 CityScout`

一句话：
输入一个项目目标和城市片区，Skill 自动生成“可执行的现场踏勘路线 + 观察清单 + 风险点 + 拍照点位 + 汇报摘要”。

为什么适合 Frank：

- 你的专业优势明显，别人容易做泛旅游、泛生活，你能做有行业判断的“地图 + 城市空间 + 决策支持”。
- 高德能力有存在感：地理编码、POI 检索、周边设施、路线规划、步行/驾车路线、距离时间、行政区定位。
- Demo 不必重工程：可以用 Skill 说明 + API 调用流程 + 示例输出 + 简单网页/脚本演示拿下。
- 传播标题有辨识度：`高德开放平台skill：一句话生成城市更新踏勘路线和汇报清单`。

## 不选普通旅游路线的原因

风险：官方例子已经有城市打卡、Citywalk、盲盒探索。再做旅游路线，很容易被评为“参考题变体”。

更小路径：保留“路线规划”的展示性，但把场景换成专业、高价值、低同质化的城市踏勘。

## MVP 范围

输入：

- 城市/片区：例如“福州烟台山历史风貌区”
- 任务类型：城市更新踏勘、商业选址、夜间安全、街区活力、文旅动线
- 时间预算：例如 90 分钟
- 出行方式：步行/驾车/公交

处理：

- 用高德能力定位片区
- 搜索周边 POI：地铁/公交、停车场、学校、医院、商业、景点、公园、社区服务、派出所等
- 计算候选点距离和路线顺序
- 生成踏勘路线与观察清单

输出：

- 路线名称
- 起终点和 5-8 个踏勘点
- 每个点的观察任务
- 风险/机会判断
- 拍照建议
- 3 句领导汇报摘要
- 使用教程

必须交付：

- `skill.md`
- 使用教程 PDF/Markdown
- Demo 演示文件或视频
- GitHub 仓库 README
- ClawHub 发布页
- 外站传播文章

## 非目标

- 不做完整 GIS 平台。
- 不做数据大屏。
- 不承诺自动给出规划结论。
- 不做复杂权限、账号、数据库。

## 升级路径

1. MVP：静态示例 + API 流程 + 可复制 Skill prompt。
2. Demo：网页输入城市/片区，调用高德 API 或模拟 API 输出路线。
3. 强化：加入多场景模板，形成“城市更新/商业选址/夜跑安全/亲子路线”模式切换。
4. 传播：写一篇技术社区文章，标题带 `高德开放平台skill`，附 Demo 视频/GIF。

## 成功测试

一个评委在 3 分钟内能回答：

- 这个 Skill 解决什么真实问题？
- 它哪里用了高德开放平台？
- 我能不能马上照着教程使用？
- 它和普通路线推荐有什么明显不同？
- 我是否愿意收藏这个 GitHub 仓库，或在 ClawHub 下载试用？

## 备选方向

1. `夜行安全雷达`
   - 优点：大众易懂，演示强。
   - 风险：官方灵感已有夜跑安全卫士，容易同质化。

2. `县域文旅一日策展人`
   - 优点：适合文旅、城市传播，有专业故事线。
   - 风险：容易变成旅游推荐，需要强 Demo。

3. `低碳通勤谈判官`
   - 优点：AI + 地图 + 个人效率，有传播性。
   - 风险：与高德能力结合要做扎实，否则像普通通勤规划。
