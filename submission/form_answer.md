# 报名表填写稿

## Skill 名称

城市踏勘官 CityScout

## 使用场景

城市更新、商业选址、文旅动线、街区活力评估等工作经常需要先做现场踏勘，但传统做法依赖人工查地图、找 POI、排路线、列观察清单，容易遗漏关键点，也很难快速形成可汇报的现场行动方案。

城市踏勘官 CityScout 面向规划师、设计管理人员、招商运营人员、文旅策划人员和政府项目工作人员。用户只需要输入城市片区、任务目标、时间预算和出行方式，Skill 就会结合高德开放平台的地理编码、POI 搜索、周边搜索、路线规划等能力，生成一份可执行的现场证据路线和观察清单。

典型场景：

- 城市更新项目前，快速形成 60-120 分钟现场证据路线。
- 商业选址前，对片区交通、商业、公共服务和竞品分布做初步判断。
- 文旅线路策划前，把景点、商业、交通和公共空间组织成可体验路线。
- 街区安全或夜间活力评估前，规划重点观察点和风险核查点。

## 功能描述

城市踏勘官 CityScout 是一个“AI + 地图 + 现场调研”的创意 Skill，核心叙事是“唤醒城市沉睡的基因”。它把用户的自然语言需求转化为一套面向现场执行的路线与调研包。

核心功能：

1. 任务识别：识别用户要做城市更新、商业选址、文旅动线、街区活力或安全评估。
2. 地点定位：调用高德地理编码能力，将片区、地名或地址转成可检索的空间范围。
3. POI 搜索：围绕目标片区检索交通站点、停车场、商业、景点、公园、学校、医院、社区服务、派出所、便利店等关键设施。
4. 路线组织：调用高德路线规划能力，根据时间预算和出行方式组织 5-8 个现场点位。
5. 观察清单：为每个点位生成“为什么去、看什么、拍什么、怎么判断”的现场任务。
6. 风险与机会判断：区分地图证据、现场待核实问题和初步推断，避免把路线推荐伪装成规划结论。
7. 汇报摘要：生成 3 句可直接放进会议材料的踏勘摘要。

高德能力调用点：

- 地理编码 API：片区/地址定位。
- POI 搜索与周边搜索：查找周边设施、商业节点、安全节点和公共服务。
- 路径规划 API：按步行、驾车或公交生成可执行路线。
- 行政区划能力：补充片区所属行政区与空间背景。
- 天气能力：用于现场踏勘提醒。
- JSAPI/静态地图：用于 Demo 展示路线与点位。

与普通路线推荐不同，城市踏勘官不是“推荐去哪玩”，而是围绕一个真实城市判断唤醒线索、组织证据链、安排现场观察动作和汇报语言。
